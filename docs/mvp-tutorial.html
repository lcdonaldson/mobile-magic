Mobile Magic MVP Tutorial

This is a browser-style walkthrough to get from zero to a full styled screen in a simulator using mobile-magic.

MagicProvider useTheme tokens 10 components
Use this with an Expo app running in simulator. Keep this page open in your browser while you code.

Step 0 — Create and run a demo app

Use a packed local install (important)

For Expo 56 + local package testing, install from a tarball instead of a direct file: link. This avoids duplicate React resolution and hook-call errors. 

cd /absolute/path/to/Mobile_Magic
npm run build
npm pack

npx create-expo-app mobile-magic-demo
cd /absolute/path/to/mobile-magic-demo
npm install ../Mobile_Magic/mobile-magic-0.1.0.tgz
npx expo start
Checkpoint: simulator opens the Expo starter app.
If you install with file:../Mobile_Magic, Metro can resolve React from both the app and the library workspace, causing: Invalid hook call at MagicProvider. Use the tarball method above.

Step 1 — Wire Provider + Screen (Expo Router / SDK 56)

Expo SDK 56 projects commonly use src/app/ + _layout.tsx, not a top-level App.tsx. Do these two edits:

1A) Update src/app/_layout.tsx

import { DarkTheme, DefaultTheme, ThemeProvider } from 'expo-router';
import { useColorScheme } from 'react-native';
import { MagicProvider } from 'mobile-magic';

import { AnimatedSplashOverlay } from '@/components/animated-icon';
import AppTabs from '@/components/app-tabs';

export default function TabLayout() {
  const colorScheme = useColorScheme();
  return (
    <MagicProvider>
      <ThemeProvider value={colorScheme === 'dark' ? DarkTheme : DefaultTheme}>
        <AnimatedSplashOverlay />
        <AppTabs />
      </ThemeProvider>
    </MagicProvider>
  );
}

1B) Replace src/app/index.tsx

import React, { useState } from 'react';
import { ScrollView, StyleSheet, View } from 'react-native';
import {
  Avatar,
  Badge,
  Button,
  Card,
  Checkbox,
  Field,
  ListRow,
  ProgressBar,
  Switch,
  Type,
  spacing,
  useTheme,
  shadows,
} from 'mobile-magic';

function DemoScreen() {
  const theme = useTheme();
  const [name, setName] = useState('Levi Donaldson');
  const [email, setEmail] = useState('levi@example.com');
  const [notifications, setNotifications] = useState(true);
  const [termsAccepted, setTermsAccepted] = useState(false);
  const [progress, setProgress] = useState(0.42);

  return (
    <ScrollView
      contentContainerStyle={[
        styles.container,
        { backgroundColor: theme.bg },
      ]}
    >
      {/* Header */}
      <View style={styles.header}>
        <Avatar name={name} size="lg" skin="info" />
        <View style={{ flex: 1, gap: spacing.xs }}>
          <Type scale="h2">Profile Setup</Type>
          <Type muted>Build a complete styled screen with Mobile Magic</Type>
        </View>
        <Badge skin="success">MVP</Badge>
      </View>

      {/* Progress */}
      <Card skin="surface" style={styles.card}>
        <Type scale="label" muted>Completion</Type>
        <Type scale="h3">{Math.round(progress * 100)}%</Type>
        <ProgressBar value={progress} skin="primary" style={{ marginTop: spacing.sm }} />
      </Card>

      {/* Form */}
      <Card style={styles.card}>
        <Type scale="h3">Account Details</Type>
        <View style={styles.stack}>
          <Field
            label="Full Name"
            value={name}
            onChangeText={setName}
            placeholder="Your name"
          />
          <Field
            label="Email"
            value={email}
            onChangeText={setEmail}
            keyboardType="email-address"
            placeholder="you@example.com"
          />
          <Checkbox
            checked={termsAccepted}
            onCheckedChange={setTermsAccepted}
          >
            I accept the terms and privacy policy
          </Checkbox>
        </View>
      </Card>

      {/* Settings rows */}
      <View style={styles.stack}>
        <ListRow
          label="Notifications"
          sublabel="Push alerts for account activity"
          leading={<Badge skin="info">N</Badge>}
          trailing={
            <Switch
              value={notifications}
              onValueChange={setNotifications}
              skin="success"
            />
          }
        />

        <ListRow
          label="Theme"
          sublabel="Follows system dark mode automatically"
          leading={<Badge skin="secondary">Auto</Badge>}
          trailing={<Badge skin="primary">System</Badge>}
        />
      </View>

      {/* Actions */}
      <View style={styles.actions}>
        <Button skin="secondary" onPress={() => setProgress(0.1)}>
          Reset
        </Button>
        <Button
          skin="primary"
          onPress={() => setProgress((p) => Math.min(1, p + 0.15))}
        >
          Continue
        </Button>
      </View>

      <Button
        skin="danger"
        disabled={!termsAccepted}
        onPress={() => {}}
      >
        Complete Setup
      </Button>
    </ScrollView>
  );
}

export default function HomeScreen() {
  return <DemoScreen />;
}

const styles = StyleSheet.create({
  container: {
    padding: spacing.lg,
    gap: spacing.md,
    minHeight: '100%',
  },
  header: {
    flexDirection: 'row',
    alignItems: 'center',
    gap: spacing.md,
  },
  card: {
    gap: spacing.sm,
    ...shadows.sm,
  },
  stack: {
    gap: spacing.sm,
  },
  actions: {
    flexDirection: 'row',
    gap: spacing.sm,
  },
});
Checkpoint: saving both files should render the new screen in simulator.

Step 2 — Verify in simulator

You should now see:

  • A header with avatar, title/subtitle, and a status badge
  • A progress card with animated linear progress
  • Form fields + checkbox agreement row
  • Two list rows with leading/trailing slots
  • Primary/secondary/danger buttons
  • Dark mode switching automatically with device theme
Checkpoint: if this renders correctly, you now understand the core usage model.

Step 3 — Understand the pattern you just used

  • MagicProvider is mounted once at app root.
  • useTheme() gives ambient colors for custom layout code.
  • Components use skin for color identity.
  • Tokens (spacing, radii, type, shadows) control structure, not theme colors.

Step 4 — Next practical tweaks

  • Swap button skins and see global tone changes.
  • Try custom raw skin object on one component.
  • Force mode with <MagicProvider mode="dark"> for testing.
  • Duplicate the screen to create a second route with same system patterns.
